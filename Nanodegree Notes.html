Web Basics

The World Wide Web was invented in the 1990s and is made up of over 30 billion pages. Those pages are HTML documents. HTML stands for HyperText Markup Language. Those documents are linked together using HyperLinks. Within the HTML documents there can be many types of files that can be accessed and downloaded which include, HTML, Plain Text, Images, Videos, and Music.

Major pieces of the web

You, your computer and browser, the internet, and servers. Your computer sends request via the internet to servers. Those requests are sent using a protocol called http. The server responds with files that you r browser displays.

HTML - HyperText Markup Language

HTML is made up of:
  • text content - what you actually see
  • markup - what the page looks like (style)
  • references to other documents - images, videos
  • links to other pages

HTML Markup:

Tags - An HTML tag is always contained within angled brackets. An example of this would be <name> contents </name> Most tags have an opening tag (<p> for example) and a closing tag, ( </p>). Some tags (called "void" tags) do not require a closing tag (like the <br> tag).
Element: An HTML element refers to everything within a set of opening and closing tags.
Attribute: This is a property of an HTML element.

Example Tags:

  • Bold - <b> contents </b>
  • Emphasis - <em> contents </em>
  • Break - <br> adds a line break
  • Paragraph - <p> contents </p>

HTML Attributes:

<TAG ATTR = "value"> contents </TAG>

  • Anchor Tag - <a href="http://www.udacity.com">Udacity</a>
  • Image tag - <img> <img scr = "url" alt = "text">
Note: Image tags do not have a closing tag. These types of tag are called Void Tags

Inline and Block elements

An Inline element adds a new line to the end of your text. A Block element adds a text box around your text. Block elements have height and width dimensions to them. The <br>, <em>, and the <img> tag are examples of an Inline element and the <p>text</p> is an example of a Block element.

Creating A Structured HTML Document

HTML is considered the structure of the web page. HTML is the basic language of the Web. Like any language it t has a specific syntax and rules. HTML uses Tags to define the structure of the web page. The browser interprets HTML tags into elements that form a tree. The browser knows how to do this because of the DOM, (Document Object Model). The DOM is a standard convention for representing and interacting with elements in HTML. Each HTML tag creates an element in the DOM that the browser uses to display the page.

Everything in an HTML document is an element. These elements have hierarchical relationship to each other. They can be parent, child or sibling. The top element is the root element. Every element has exactly one parent except the root element. An element can have a number of children. Sibling elements are elements with the same parent.

Adding CSS Style and HTML Structure Subtitles

CSS or Cascading Style Sheets. You can have several CSS files or sheets that will be referenced an HTML page. With CSS the most specific rule is applied to every element. The concept of most specific rule applied is what the word cascading means. You keep going down and down into the rules until you find the one that best describes the elements. Then that rule gets applied. Changing the visual properties of an element only changes how the element is displayed, not what kind of element it is. Some examples of style tags are H1 and H2, H3. CSS can be used to position elements on the web page. One example is the flex box layout. Flex box means flexible box and it provides an efficient way to lay out, align, and distribute space among items in a container or div.

A web page is a collection of boxes, in which each box contains 4 spatial components content, padding (to separate the content from the border), border, and margin (to separate the border from the adjacent boxes) . The size of the box can be set by by % or by pixel. These attributes can all be set within the HTML code or inside a CSS
